OS X系统下配置Sublime Text 3支持C++11

OS X系统下面，Sublime无疑是最好用的编辑器，Sublime已经推出了Sublime Text 3（ST3）,官网介绍说Sublime Text 3 makes extensive use of functionality in C++11。

在Sublime Text 3下面，C++的build system默认调用的是g++。要想在Mac OS X 10.8—Mountain Lion 系统下让Sublime Text 3可以直接编译支持C++11，有两种办法。

  1. 通过homebrew升级g++版本，然后修改C++的build的配置参数
  2. 换用clang++（前提是安装好了xcode）

明显第二种方式简单省事。

  1. 打开Sublime，新建一个build-system。

    Tools->Build System->New Build System

  2. 编写配置文件如下

    {
"cmd": ["clang++", "${file}","-std=c++11", "-stdlib=libc++", "-o", "${file_path}/${file_base_name}"],
"file_regex": "^(..[^:]*):([0-9]+):?([0-9]+)?:? (.*)$",
"working_dir": "${file_path}",
"selector": "source.c, source.c++",


"variants":
[
    {
        "name": "Run",
        "cmd": ["bash", "-c", "clang++  '${file}' -std=c++11 -stdlib=libc++ -o '${file_path}/${file_base_name}' && '${file_path}/${file_base_name}'"]
    }
]
}

  3. 保存，可以保存名字为C++11.sublime-build,保存在默认目录下。

  4. Done!Build System保持默认Automatic，这样在Sublime Text 3里面写完程序，直接通过按下CMD + B就可以进行编译了。

由于配置文件里面已经通过select指定了文件类型，因此在Build System还是选择Automatic,当我们通过CMD + BSublime会使用我们保存的C++11.sublime-build配置进行build，非常方便，可以通过CMD + shift + B直接运行程序。

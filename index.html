OS X Yosemite(10.10)下无法使用git svn解决方法 2014-11-06

自己在OS X下面没有找到好用的免费SVN客户端，又不想使用破解软件，更因为觉得SourceTree实在是太好，对它爱不离手，因此当自己遇到代码管理使用SVN的时候，还是习惯使用git svn做一个桥接，让代码在我本地是通过GIT来管理的。

近日电脑系统升级到了Yosemite后，使用git svn clone命令报错，错误信息如下：
Can’t locate SVN/Core.pm in @INC (you may need to install the SVN::Core module)

添加Perl库链接

这个错误以前在10.9的时候也遇到过，Google一下就找到了解决方案。那个时候Perl库用的是5.16版本，现在系统升级到了5.18版本后，跟以前一样，添加两条软连接命令：

sudo ln -s /Applications/Xcode.app/Contents/Developer/Library/Perl/5.18/darwin-thread-multi-2level/SVN /System/Library/Perl/Extras/5.18/SVN

sudo ln -s /Applications/Xcode.app/Contents/Developer/Library/Perl/5.18/darwin-thread-multi-2level/auto/SVN/ /System/Library/Perl/Extras/5.18/auto/SVN

如果提示你找不到相关的原始文件，需要安装下command line tool，通过命令：xcode-select --install即可安装。

青春不过几届世界杯 2014-07-15

决赛

德国夺冠了，不枉他们连续多年打入四强，终于在三驾马车后，又一次捧起了大力神杯。决赛是凌晨3点开始的，我还报名参加了公司集体看球活动。前一晚在公司调代码，后面11点多的时候感到太困，就想着先回家睡一觉再来公司看吧，毕竟家到公司走路3分钟就搞定了。回到家，直接躺下睡，我还不枉给fitbit和手机设了两个闹钟，可是当我醒过来的时候，已经是4点了，打开手机，看到下半场刚开始，可是我也没有爬起来跑去公司看，选择了翻个身继续睡。

98法国世界杯

关于世界杯的记忆，可以追述到1998年，那个时候我还只是个9岁的小孩，虽然没有看直播，我还记得那个时候是在袁媛家里看到的重播。那个时候读小学，因为经常在袁媛家等老妈来接我，因此有机会在她家看电视。通过电视，虽然看到了巴西的0:3,但是让我知道了罗纳尔多，更让我从此成为了一名国米球迷，即使罗纳尔多出走后，我还是喜欢着这家俱乐部。因为罗纳尔多爱上国米，因为莫拉蒂，所以对国米不离不弃。

02韩日世界杯

2002韩日世界杯，我已经上初中，记得是在西山的家里看的决赛直播，看着决赛罗纳尔多的两 粒进球，卡福举起大力神杯，站到了颁奖台上面。那届的世界杯，更因为中国队的缘故，国内关注度盛况空前。中国队的比赛，怂恿老师打开班级里面的那台电视机，看上几十分钟的中国队比赛。那年的巴西，性感闪耀，再回头看，那届巴西应该是最近20年最完美的巴西队。如今的巴西队，应该只有内马尔能够跻身主力阵容。

06德国世界杯

2006年德国世界杯，我已经上高中了。由于时差的原因，比赛都是半夜。那个时候在木渎中学寄宿上学，为了能够看到世界杯，我跑到外面修电视的地方，买了一个二手的那种很小的电视机装在书包里藏到了寝室里。由于寝室限电，插座没有电，万幸的是那个时候天气热了，给我们开了空调，只有空调那个插座有电，借助陈龙飞的拖线板，我们偷偷给电视机供上了电。借助天线，我们收到了央视一套的信号，这是那个电视机能够收到的两个电台中的一个。就这样，我们在高中寝室里半夜偷偷的看起了世界杯。到了半夜，别的寝室的，会来偷偷敲门，我们知道是来看球的小伙伴。我们看到巴西打澳大利亚，期待小罗，卡卡，罗比尼奥，罗纳尔多，阿德里亚诺能够打出一场精彩的比赛，可惜这些球星都不在状态。看着罗纳尔多被换下，王恒很失望，没有了大罗的巴西，他都没有兴趣继续看下去了。我们看阿根廷的比赛看到天亮，看到马克西·罗德里格斯那脚神来之笔绝杀，那个兴奋。当然也目睹了巴西被法国淘汰出局，还记得当时心脏的那个紧张的感觉，当最后泽·罗伯托的射门偏出之后，知道五星巴西出局了。那种紧张的感觉，当若干年后，我在天河球场，亲眼目睹恒大夺得亚冠的时候，再度袭来，只是这次不是落后，还是希望能够守住这个比分，当时感觉时间过的太漫长了。那年的时间杯，三喵军团特别的惨，打葡萄牙鲁尼被红牌罚下，杰拉德，兰帕德点球决战射失点球。最终，随着神奇的格罗索在点球大战中命中，世界杯在意大利的夺冠中落幕。

Get busy living, or get busy dying 2014-05-20

我想我总该写点什么了，自己欠自己的东西越来越多。年前的时候就想把11月份的游记整理好，可是到了现在自己都没有动笔记录下来，半年都已经过去了，如果没有那些照片，自己真的以为那只是自己的一场梦，一场难忘的美梦。自己承诺给自己建的小型气象站也没有弄，太多的事情都没有去实现，比起那个曾经说走就走的自己，那个可以在足球场踢一天足球的自己，那个可以花一天时间从玉泉骑到家里的自己，像完全变了一个人似的。

我似乎开始变得沉默寡言，喜欢把更多的事情藏在心里，翻开自己的日记本，最后一篇日记的时间是2013-05-15，竟然已经是一年多以前了。这一年发生了太多的事情，找工作，实习，旅行，毕业，工作，我都没有在日记里面记录下来。自己以前喜欢翻阅自己那些年的日记，总觉得那个时候的自己是多么的很傻很天真，这一年没有写日记，也有一定原因是不喜欢以后的自己看到现在傻傻的我。

周五那天看着大家的周报，突然感觉自己应该写点什么，让自己知道自己不应该这样虚度光阴，把最美好的岁月就这样浪费了。在Evernote里面建了一个笔记本，用来记录自己每天干的事情，总结下自己每天的学习与收获，知道自己一天不是在混混沌沌中度过的。想着给这个笔记本取个什么名字比较好，突然肖申克救赎里面的那句名言在我脑海里面一闪而过：Get busy living, or get busy dying。

过去的自己就好像坐在自己身边，对我说着：These walls are kind of funny like that. First you hate them, then you get used to them. Enough time passed, get so you depend on them. Thats institutionalized。

不想再多说什么了，就用5月9号公司在黄龙体育馆的年会的那条标语来结尾——“唯有梦想与好姑娘不可辜负”！

利用Xively平台展示气象数据 2013-09-07

利用Xively物联网平台，我的blog里面已经可以展示我阳台的气象数据了。Xively网站上可以看详细数据,Yeelink有更多历史数据。

这些数据是由Arduino完成的采集，然后通过Zigbee网络传输到了Raspberry Pi，Raspberry Pi再将这些采集的数据传输到了Xively。

传送到Xively

Xively网站提供了POST接口供我们传输数据，但同时官网提供了不同语言下的SDK，这样传输数据非常方便。
首先，我们需要先在Xively网站申请注册一个账号，然后注册一个设备，获得两个关键的参数API_KEY和Feed ID。其中API_KEY是用来完成用户验证的，Feed ID则用来确定传输数据的目的地，也就是API接口地址。 在每一个FEED下面，我们可以添加不同的channel来接收显示不同的数据。
以官网的基于Raspberry Pi教程为例，使用python SDK完成数据传输。
首先需安装python SDK

Raspberry Pi下使用MongoDB 2013-08-31

其实我更想把标题改为Raspberry Pi下折腾MongoDB。在开始折腾前，先介绍一个linux下的工具。

安装准备

在很多场景下，我都是都是SSH登录到Linux主机进行操作的，而不是直接在Linux主机上面通过键盘操作。一旦我们断开连接，如果任务没有执行完，那这个我们在这个SSH下发起的任务都会被终止掉，这个时候我们借助screen可以很好的帮我们把这个问题解决掉。关于screen，推荐阅读下《Linux技巧：使用screen管理你的远程会话》,详细介绍了为什么会关闭，到怎么使用screen。
Raspberry Pi官方提供的镜像下面没有自带screen，因此需要我们先安装下。

sudo apt-get install screen

安装MongoDB

由于没有官方镜像提供，我们选择以源码方式安装。

  1. 相关工具安装

    sudo apt-get install build-essential libboost-filesystem-dev libboost-program-options-dev libboost-system-dev libboost-thread-dev scons libboost-all-dev git

  2. 获取源代码
    MongoDB的源代码是基于x86机器的little endian的，Raspberry Pi的CPU是基于ARM的big endian，在github上面有big endian的nonx86版本提供。

Dating

这道题目其实有歧义，表述上面有问题。

出题的意思是先比较前面两个string找到日期，然后找到小时，然后比较后面两个string，找到分钟，最后控制输出格式。

void dating(string a,string b,string c,string d)
{
    int day = -1;
    int hour = -1;
    int min = -1;
    for(int i=0;i<a.size();i++)
    {
        if (a[i] == b[i]) 
        {
            if (day == -1)
            {
                if(a[i] >= 'A' && a[i] <= 'G')
                {
                    day = a[i] - 'A';
                }
            }
            else if (hour == -1)
            {
                if(a[i] >= 'A' && a[i] <= 'N')
                {
                    hour = a[i] - 'A' + 10;
                    break;
                }
                else if (isdigit(a[i]))
                {
                    hour = a[i] - '0';
                    break;
                }
            }
        }
    }

    string Week[] = {"MON", "TUE", "WED", "THU", "FRI","SAT","SUN"};
    for(int i=0;i<c.size();i++)
    {
        if ( (c[i] == d[i]) && (isalpha(c[i])) )
        {
            min = i;
            break;
        }

    }
    cout.fill('0');
    cout << Week[day] << " " << setw(2) << hour << ":" << setw(2) << min << endl;
    return;

}
